# 어제이슈 출시 다음 단계

## 지금 상태

- 제출용 파일: `app/eojje-issue.ait`
- 뉴스 데이터: 30건
- 카테고리: 정치/경제/사회/문화/연예 각 6건
- AI: 6건 미만인 날에는 카테고리와 탭을 숨김
- 빈 요약: 0건
- 제목-요약 불일치: 0건
- 약한 요약 의심: 0건
- OpenAI 키: 출시 전에는 연결하지 않음
- 광고: 실제 App in Toss 광고 SDK 연결 전까지 제출 빌드에서는 숨김

## 왜 이렇게 하나

광고 자리는 수익화를 위해 필요하지만, 실제 광고 SDK 없이 `AD` 미리보기 영역을 제출하면 미완성 앱처럼 보일 수 있다. 그래서 개발 중에는 확인할 수 있게 두고, 제출용 빌드에서는 숨긴다.

OpenAI 키는 지금 넣지 않는다. 현재 요약은 기사 원문 기반 추출형이라 심사 리스크가 낮고, 생성형 AI 결과물 고지 의무를 피할 수 있다.

## App in Toss 콘솔에서 해야 할 일

1. 앱 정보 확인
   - 앱 이름: 어제 이슈
   - 브랜드 컬러: `#191F28`
   - 아이콘: `app/public/icon.svg`

2. 앱 기능 등록
   - 비게임 앱은 최소 1개 앱 기능이 필요하다.
   - 추천 기능명: `어제 이슈 브리핑 보기`
   - 설명: `어제 많이 이야기된 뉴스를 오늘 아침 빠르게 확인하는 기능`

3. `.ait` 업로드
   - 파일: `app/eojje-issue.ait`

4. 테스트
   - 홈 화면 진입
   - 기사 상세 열기
   - 저장 버튼
   - 원본 뉴스 보기 링크
   - AI 카테고리는 데이터가 6건 이상일 때만 노출
   - 모든 상세 첫 장이 `30초 브리핑`으로 시작하는지
   - `사람들 반응`, `주의할 점` 카드가 남아 있지 않은지
   - `app` 폴더에서 `npm run check` 통과
   - `app` 폴더에서 `npm run build` 통과

5. 심사 요청 전 확인
   - 외부 링크가 앱 핵심 기능을 대체하지 않는지 확인
   - 광고 SDK 없이 광고 미리보기 영역이 보이지 않는지 확인
   - 생성형 AI 결과물 표시가 필요한 기능이 없는지 확인
   - 원격 데이터가 최신 기준인지 확인: `pipeline` 폴더에서 `npm run validate:remote`
   - GitHub Actions `ci`와 `daily-news`가 모두 통과하는지 확인
   - `npm audit --omit=dev` 결과를 확인하고, App in Toss SDK 하위 의존성 리스크를 수용 가능한지 판단
