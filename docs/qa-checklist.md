# 어제 이슈 - QA 체크리스트

## 데이터 (매 발행마다 자동 검사됨 - quality-report.md 확인)

- [ ] 헤드라인은 활성 카테고리별 1개씩. AI가 있으면 6개, AI가 없으면 5개
- [ ] 활성 카테고리당 6개 이상. 단, AI는 업데이트 소식이 부족하면 생략 가능
- [ ] 제목/요약이 `...`으로 끝나는 항목 0개 (있으면 발행 차단)
- [ ] 사진 캡션/파편/run-on 요약 0개
- [ ] 같은 인물·사건 중복 0쌍
- [ ] 모든 기사 publishedAt이 40시간 이내
- [ ] 출처명이 실제 언론사명인지 (예: "co", "출처 미상" 비율 확인)
- [ ] 민감도 "높음" 기사가 헤드라인에 없는지
- [ ] 보도자료성/인사·동정/지역 홍보성 기사가 상위에 없는지 (육안)

## 앱 기능

- [ ] 첫 진입: 스켈레톤 → 헤드라인 렌더
- [ ] 탭 전환 (전체/정치/경제/사회/문화/연예/AI/저장). AI 데이터가 없으면 AI 탭은 비어도 앱이 깨지지 않아야 함
- [ ] `app` 폴더에서 `npm run verify:data`, `npm run verify:ui`, `npm run verify:story` 통과
- [ ] 카드 탭 → 바텀시트 열림, ×/백드롭/ESC로 닫힘
- [ ] 시트 내 "원본 뉴스 보기" → 새 창으로 원문 열림
- [ ] 저장 토글: 카드/시트 양쪽에서 동작, 상단 카운트 갱신
- [ ] 저장 후 앱 재시작 → 저장 목록 유지 (localStorage)
- [ ] 다음 날 데이터가 바뀌어도 저장 목록의 기사 열람 가능 (스냅샷)
- [ ] 데이터 2일 이상 경과 → "최신 뉴스 갱신에 실패했어요" 배너
- [ ] 네트워크 차단 + 번들 데이터 없음 → 오류 화면 + 다시 시도 동작
- [ ] 요약 없는 기사 상세: "요약이 제공되지 않는 기사예요" 안내

## 화면 품질

- [ ] 375px 너비에서 텍스트 중간 잘림 없음
- [ ] 제목/요약 전문 노출 (말줄임 없음)
- [ ] 저장 버튼 터치 영역 40px 이상
- [ ] 카테고리 색상 구분 (정치 빨강/경제 초록/사회 파랑/문화 보라/연예 주황)
- [ ] 민감도 중간/높음 기사 상세에 원문 확인 안내 문구

## 심사 리스크 점검 (출시 전 1회 + 주기적)

- [ ] 정치 기사: 특정 정당/인물 비방·옹호로 읽히는 요약 없는지 (요약은 원문 추출 문장만 사용)
- [ ] 연예 기사: 루머/사생활 단정 표현 없는지, 민감도 중간 이상 분류 확인
- [ ] 모든 기사에 출처명+시간+원문 링크 표기
- [ ] 기사 전문 저장/복사 없음 (제목+요약 1~2문장만)
- [ ] 앱이 자체 의견/예측/투자조언을 내지 않음
